※Flyway使用

V0  : 前提条件（ユーザー作成、拡張機能インストールなど）  
V1  : 基本スキーマ作成（テーブル、インデックス）  
V2  : 追加スキーマ作成  
V3  : マスターデータ挿入  
V4  : 制約追加（外部キー、CHECK制約など）  
V5  : ビュー作成  
V6  : ストアドプロシージャ/関数作成  
V7  : 権限付与（最後）  
V8+ : 継続的な変更


## 理由
### 1. 依存関係の順序
ユーザー → テーブル → データ → 制約 → 権限

### 2. エラー時のロールバック
テーブル作成が失敗したら、そこで止まる
権限付与は最後なので、本質的なスキーマは完成している

### 3. 環境の違いへの対応
開発環境では権限付与をスキップできる
本番環境では全て実行
