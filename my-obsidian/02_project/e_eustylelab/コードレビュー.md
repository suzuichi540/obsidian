
GitHub でコードレビューする際のガイドラインです。

## コメントタグ

[](https://github.com/eustylelab/readme/blob/master/docs/guidelines/code-review-guideline.md#%E3%82%B3%E3%83%A1%E3%83%B3%E3%83%88%E3%82%BF%E3%82%B0)

- 《推奨》コードレビューでコメントする際、先頭に必ず次のいずれかのタグをつけてください。

|タグ|意味|
|:--|:--|
|`[MUST]`|必ず修正して欲しい内容をコメントする場合。|
|`[MAY]`|できれば修正して欲しい内容をコメントする場合。|
|`[IMO]`|個人的な好みや提案をコメントする場合（In My Opinion の略）。|
|`[FYI]`|豆知識やプチ情報などをコメントする場合（For Your Information の略）。|
|`[ASK]`|質問をコメントする場合。|

### 例

[](https://github.com/eustylelab/readme/blob/master/docs/guidelines/code-review-guideline.md#%E4%BE%8B)

```
[MUST]
コーディングガイドライン違反です。変数名は camelCase としてください。
```

```
[MAY]
スペルミスがあるのでできれば修正してください。
```

```
[IMO]
改行位置がバラバラで読みづらいように感じます。以下のようにしてみてはどうでしょうか。

...
```

```
[FYI]
今回はこれでもいいかと思いますが、もっと便利な `awesome` という関数があります。

詳しくはこちら -> https://www.example.com/...
```

```
[ASK]
ここで●●をしている理由がわかりませんでした。教えてください。
```
