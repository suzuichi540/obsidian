
Git のコミットコメントを記述する際のガイドラインです。

## 全般

[](https://github.com/eustylelab/readme/blob/master/docs/guidelines/commit-message-guideline.md#%E5%85%A8%E8%88%AC)

- 《必須》コミットで「何が起こるか」「何をしたか」ではなく「どう変わるのか」を記述してください。
- 《必須》日本語、または英語で記述してください。
- 《推奨》他人（または数ヶ月後、数年後の自分）が読んで、理解できる内容とすることを心掛けてください。

## 1行目

[](https://github.com/eustylelab/readme/blob/master/docs/guidelines/commit-message-guideline.md#1%E8%A1%8C%E7%9B%AE)

- 《必須》コミットに関する要約、概要を記述します。半角50文字（全角25文字）程度を目安としてください。
- 《必須》句読点は使わないこと。
- 《必須》末尾に半角スペース＋角括弧で囲った Backlog の課題番号を記述してください。
- 《必須》先頭にコミットの内容がわかる絵文字（後述）を必ず入れてください。 ただし直接絵文字を入力せず `:sparkles:` のような GitHub 互換 Markup で入力してください。
- 《必須》過去形、過去完了系、受動態は用いず、現在時制を用いて記述してください。命令形でも構いません。
    - × 新しい機能を追加した
    - × 新しい機能が追加された
    - ○ 新しい機能を追加する
    - ○ 新しい機能を追加

## 2行目

[](https://github.com/eustylelab/readme/blob/master/docs/guidelines/commit-message-guideline.md#2%E8%A1%8C%E7%9B%AE)

- 必ず空行としてください。

## 3行目以降

[](https://github.com/eustylelab/readme/blob/master/docs/guidelines/commit-message-guideline.md#3%E8%A1%8C%E7%9B%AE%E4%BB%A5%E9%99%8D)

- 《推奨》1行目に収まりきらない詳細な修正内容を記述してください。
- 《推奨》コードから明確でない変更（特殊なテクニックを用いた場合など）は詳細に解説してください。
    - もっともその場合、コード中にもコメントを入れるべきです。
- 《推奨》参照するべきURL等がある場合も記述してください。
- 《推奨》箇条書きを用いても構いません。ただし「・」や「●」などは用いず、Markdown 形式に準拠するようにしてください。
- 《推奨》過去のコミットと関連する場合などは、そのコミット番号に言及するようにしてください。
- 《推奨》不具合修正の場合は、不具合の原因と解決方法を記述してください。

## 絵文字について

[](https://github.com/eustylelab/readme/blob/master/docs/guidelines/commit-message-guideline.md#%E7%B5%B5%E6%96%87%E5%AD%97%E3%81%AB%E3%81%A4%E3%81%84%E3%81%A6)

コミットの内容に応じて下記の中から適切な絵文字を選択してください。

|絵文字|Markup|コミットの内容|
|:--|:--|:--|
|🎉|`:tada:`|記念すべきコミット（初回コミットや、大きな新機能の追加など）。|
|✨|`:sparkles:`|新機能追加。|
|🐛|`:bug:`|バグ修正。|
|😄|`:smile:`|見た目やユーザービリティの改善を主目的とするコミット。|
|🎶|`:notes:`|DX（Development Experience）関連のコミット。|
|⚡️|`:zap:`|パフォーマンス改善。|
|🎨|`:art:`|コードの美化や整理。コードスタイル関連、可読性の向上を目的とした修正。|
|♻️|`:recycle:`|リファクタリング。`:art:` とうまく使い分けること。|
|🚚|`:truck:`|ファイルやディレクトリの移動、またはファイル名（ディレクトリ名）の変更。|
|🔥|`:fire:`|ファイルやコードの削除。削除に伴う関連モジュールの修正を含めてOKです。|
|📝|`:memo:`|ドキュメントやコード中のコメントの追加・編集・削除。|
|✅|`:white_check_mark:`|テスト関連のコミット。|
|🚀|`:rocket:`|デプロイ関連のコミット。|
|🔧|`:wrench:`|設定の追加・変更。|
|➕|`:heavy_plus_sign:`|依存パッケージの追加。|
|➖|`:heavy_minus_sign:`|依存パッケージの削除。|
|⬆️|`:arrow_up:`|依存パッケージ等のバージョンアップ。|
|⬇️|`:arrow_down:`|依存パッケージ等のバージョンダウン。|
|🔀|`:twisted_rightwards_arrows:`|マージ。|
|⏪|`:rewind:`|リバートコミット。|

## 例

[](https://github.com/eustylelab/readme/blob/master/docs/guidelines/commit-message-guideline.md#%E4%BE%8B)

```
:sparkles: 利用者一覧に列「年齢」を追加 [DEV-1234]

年齢の算出に特殊なロジックを使用しています（詳しくはコード中のコメント参照）

参考URL
https://www.example.com/
```

```
:bug: 利用者一覧における氏名の文字化けを解消 [DEV-8888]

以前の修正（abcd1234）において導入したモジュールが
utf-8 ではなく sjis を前提としていることがわかった

モジュール呼び出し前後に文字コード変換処理を追加することで対応する
```
