## このガイドラインについて

[](https://github.com/eustylelab/readme/blob/master/docs/guidelines/coding-style-guideline.md#%E3%81%93%E3%81%AE%E3%82%AC%E3%82%A4%E3%83%89%E3%83%A9%E3%82%A4%E3%83%B3%E3%81%AB%E3%81%A4%E3%81%84%E3%81%A6)

- 《必須》開発チームメンバーは本ガイドラインをよく読み、実践する必要があります。疑問がある場合はガイドライン担当者に確認してください。
- 《必須》ガイドライン担当者は開発チームメンバーの疑問を解き、必要な場合は共にガイドの改善策を考えなくてはなりません。
- 《必須》ガイドラインおよびガイドライン担当者は変更される場合があります。 変更された場合、ガイドライン担当者は全開発チームメンバーに変更内容とその理由を通知する必要があります。

## 各言語共通のルール

[](https://github.com/eustylelab/readme/blob/master/docs/guidelines/coding-style-guideline.md#%E5%90%84%E8%A8%80%E8%AA%9E%E5%85%B1%E9%80%9A%E3%81%AE%E3%83%AB%E3%83%BC%E3%83%AB)

- 《必須》各種名称（ファイル名、名前空間名、クラス名、メソッド名、関数名、変数名、定数名 etc...）には正確な英語を用いる必要があります。 つまり、略語および日本語（またはそのローマ字表記）は使用できません。ただし下記例外を除きます。
    - 既存のデータベース定義に合わせる場合。
    - 日本語の訳語として適切な英語がなく、やむなく日本語のローマ字表記を用いる場合。
    - あらかじめ辞書（略語一覧）を作成・共有し、それに記載がある略語を用いる場合。

## PHP

[](https://github.com/eustylelab/readme/blob/master/docs/guidelines/coding-style-guideline.md#php)

- 《必須》基本的なコーディングスタイルは PSR-1 及び PSR-2 に準拠する必要があります。
- 《必須》クラス名、名前空間は PSR-4 に準拠する必要があります。
- 《必須》グローバル変数およびグローバル関数を使ってはいけません。
- 《必須》メソッドや関数を定義する際、引数や戻り値の型宣言を常に行う必要があります。
- 《必須》変数名は camelCase である必要があります。
- 《必須》ハンガリアン記法は使用してはいけません。
- 《必須》配列リテラルの記法は古典的な `array(...)` ではなく短縮構文 `[...]` を用いる必要があります。
- 《必須》文字列リテラルの記述は原則としてシングルクォートを用いる必要があります。 ただし変数展開を用いる場合などはもちろんダブルクォートを用いて構いません。
- 《必須》可能な限り php-cs-fixer を用いることとします。

## HTML

[](https://github.com/eustylelab/readme/blob/master/docs/guidelines/coding-style-guideline.md#html)

- 《必須》文字コードは BOM なしの UTF-8 である必要があります。
- 《必須》改行コードは LF である必要があります。
- 《必須》インデントは半角空白文字4つである必要があります。タブ文字は使ってはいけません。
- 《必須》特段の理由がない限り DOCTYPE は HTML5 である必要があります。XHTML は使ってはいけません。
- 《必須》要素名や属性名にアルファベットの大文字を使うことを禁止します。
- 《必須》属性値を記述する場合は、ダブルクォートを使用する必要があります。
- 《必須》属性値を複数記述する場合は、それぞれの間に単一の空白か、改行（とそれに続くインデント）を挿入する必要があります。 また、改行で区切る場合は、すべての属性がそれぞれ単一の行に記述される必要があります。
- 《必須》HTML5 を採用できない場合を除き、スタイルシートとスクリプトの type 属性は省略する必要があります。
- 《必須》div などのブロック要素は（例えスタイルシートでインライン要素として表示しようとしていたとしても） 常に新しい行に配置する必要があります。インライン要素は同一行のままで構いません。また、ブロック内での改行は自由です。

HTMLに関するルールは各種 HTML テンプレートにも適用されます。 ただし、テンプレートの記述ルールと本ルールが衝突する場合は、テンプレートの記述ルールを優先することとします。

## Stylesheet

[](https://github.com/eustylelab/readme/blob/master/docs/guidelines/coding-style-guideline.md#stylesheet)

- 《必須》文字コードは BOM なしの UTF-8 である必要があります。
- 《必須》改行コードは LF である必要があります。
- 《必須》インデントは半角空白文字4つである必要があります。タブ文字は使ってはいけません。
- 《必須》id 名および class 名は kebab-case である必要があります。
- 《必須》複数のプロパティを記述する際はアルファベット順に定義する必要があります。 ただしやむを得ない理由によりベンダープレフィクスを直接記述する場合はその限りではありません。
- 《必須》1行に記述するプロパティは1つである必要があります。複数のプロパティを単数行に記述してはいけません。
- 《必須》`{` の後には改行を入れる必要があります。
- 《必須》`}` の前後には改行を入れる必要があります。
- 《必須》`:` の後には半角空白を1文字のみ入れる必要があります。`:` の前には空白を入れてはいけません。

このルールは CSS のみではなく SASS, SCSS, LESS, Stylus などにも適用されます。 ただしいずれのルールについても、採用するスタイルシートフレームワークのルールと衝突する場合は、フレームワークのルールを優先することとします。

## レガシーコードの移行について

[](https://github.com/eustylelab/readme/blob/master/docs/guidelines/coding-style-guideline.md#%E3%83%AC%E3%82%AC%E3%82%B7%E3%83%BC%E3%82%B3%E3%83%BC%E3%83%89%E3%81%AE%E7%A7%BB%E8%A1%8C%E3%81%AB%E3%81%A4%E3%81%84%E3%81%A6)

レガシーコードについては、その移行途中に限り、すべてのファイル、またはファイル内すべてのコードが完全に準拠する必要はないものとします。 ただしその場合でも、それぞれのファイルの編集を行う場合は、下記の対応を行うこととします。

- 《推奨》インデントや空白、空行などコードフォーマッターによって自動整形できるものについては編集を行う前にフォーマットを行い、その後に修正するものとします。
- 《推奨》編集によって新たに記述されるコードは完全にガイドラインに準拠する必要があります。
